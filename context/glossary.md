# Glossary

## Prompt asset

A prompt treated as a reusable, revisable project artifact rather than a one-off message.

## Spec

A structured description of what should be built, why, and under what constraints.

## Constraint

A rule or boundary that limits the solution space.

## Non-goal

Something intentionally excluded from the current scope.

## Prompt version

A named revision of a prompt that can be compared against earlier runs.

## Eval

A structured review of whether the output actually met the goal.

## Run log

A record of what happened when a prompt was used, including failures and revisions.

## Benchmark

A repeatable comparison between prompt versions, models, or output styles on the same task.
