# Project Overview

## Project goal

Build a reusable prompt and spec system that improves output quality over time instead of starting from scratch on every new project.

## Who it is for

- solo builders
- product-minded engineers
- AI-heavy workflows that need consistency
- future you, who should not have to rediscover the same structure every week

## Main workflow

1. Define the task clearly.
2. Package the right context.
3. State constraints and non-goals.
4. Require a structured output.
5. Evaluate the result.
6. Log failures and revise the prompt.

## Current status

- Folder structure is in place.
- Core starter docs are scaffolded.
- Prompt templates and eval templates exist.
- The next stage is real-world use on active projects.

## What done looks like

The project is succeeding when:

- prompts are reusable instead of one-off
- specs are clear enough to hand to an AI or a human collaborator
- failures are logged and turned into revisions
- outputs are easier to compare across models and prompt versions
- new projects start from proven structure instead of blank-page chaos

## Version 1 boundary

Version 1 is a clean markdown operating system for prompts and specs.

It is not:

- a complex prompt management platform
- an automated evaluation pipeline
- a full web app
- a massive knowledge base with no maintenance plan
