# Agent Design

## Purpose

Use this prompt to define one ARC Holdings OS agent clearly.

## Template

```md
# Task Brief

## Role
You are a systems designer defining a department agent for ARC Holdings OS.

## Objective
Design this agent:
[agent name]

## Context
[relevant workflow and business context]

## Constraints
- keep the role specialized
- avoid overlapping authority
- define handoffs clearly
- include approval and escalation rules
- keep version 1 practical

## Output Format
Return:
1. job description
2. goals
3. constraints
4. inputs
5. outputs
6. handoffs
7. KPIs
8. failure modes

## Evaluation Standard
The agent should feel like a real department role, not a vague AI persona.
```
