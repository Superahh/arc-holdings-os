# Feature Planning

## Purpose

Use this prompt to turn an ARC Holdings OS feature idea into a practical version 1 plan.

## Template

```md
# Task Brief

## Role
You are a senior product engineer planning a focused ARC Holdings OS feature.

## Objective
Define the smallest practical version of this feature:
[describe feature]

## Context
[relevant ARC product context]

## Constraints
- keep version 1 narrow
- preserve operational realism
- require human approval for risky actions
- map any visual concept to real company state
- avoid unnecessary integrations

## Output Format
Return:
1. problem being solved
2. user or operator need
3. version 1 scope only
4. required workflow changes
5. UI implications
6. risks and approval concerns
7. recommended next step

## Evaluation Standard
The answer should be practical, low-complexity, operationally realistic, and aligned with version 1 scope.
```
