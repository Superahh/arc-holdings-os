# Success Criteria

A response or output is good if it:

- solves the specific task
- matches project constraints
- is practical and implementation-ready
- avoids unnecessary complexity
- is easy to revise later
- reduces ambiguity instead of adding it
- makes the next action obvious

## Repo-level success

This prompt lab is working if:

- prompts become easier to reuse
- context is easier to package
- failures are logged consistently
- revisions lead to better outputs over time
- the system stays lightweight enough to keep using
