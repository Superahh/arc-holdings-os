# Constraints

- Keep UI simple and ADHD-friendly.
- Minimize API cost.
- Avoid duplicate calls and duplicated work.
- Prefer local tools when practical.
- Preserve modular architecture.
- Do not overengineer version 1.
- Optimize for clarity over cleverness.
- Keep outputs implementation-ready.
- Favor reusable structure over one-off hacks.
- Make non-goals explicit.
- Keep the system easy to revise later.
