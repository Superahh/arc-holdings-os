# Product Spec

## Problem

Small opportunity businesses lose margin because sourcing, valuation, routing, and approvals are fragmented and inconsistent.

## User

Owner-operator who needs:

- one operational view of the business
- disciplined capital decisions
- low cognitive overhead
- safe approval control

## Core outcome

User can view a live digital company, review contract-shaped recommendations, approve consequential actions, and monitor outcomes.

## Product thesis

A CEO-led multi-agent model plus explicit contracts produces better decisions and cleaner implementation than ad hoc prompting.

## Main features (v1)

- CEO-led department agent model
- realistic opportunity workflow
- approval-aware decision gates
- top-down office simulation tied to real state
- standardized output contracts for handoffs and status
- eval and iteration loop

## Non-goals (v1)

- full autonomy for financial actions
- broad external integration coverage
- deep ERP features
- decorative game mechanics not tied to decisions

## Version 1 scope boundary

Version 1 must support:

- one realistic opportunity pipeline
- explicit agent handoffs
- `ApprovalTicket` for consequential actions
- `AgentStatusCard` plus `CompanyBoardSnapshot` for office status

## Success condition

The product is working when the user can safely make better decisions with less friction and clear visibility into what the company is doing.
